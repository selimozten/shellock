# Branding Notes

Shellock's terminal startup should stay text-first and operator-focused:

- Use one compact, centered startup card with the same rounded single-line border as the composer.
- Keep the transcript as the dominant surface. Startup is orientation, not a dashboard.
- Show only identity, version, workspace, and model in the card.
- Keep model, context, branch, and runtime in a quiet responsive footer.
- Give the composer a complete border and enough contrast to remain the primary interaction surface.
- Preserve input scroll indicators in the composer border for long prompts and pastes.
- Keep reasoning and active work visibly labeled when the selected provider exposes them.
- Use the inline Shellock name once in the startup border; avoid decorative ASCII wordmarks.
- Use a restrained accent for identity and active reasoning state, reserving success colors for actual outcomes.
- Treat user-provided logo/color work as the source of truth once available.

The visual hierarchy takes cues from Grok Build's restraint and Factory Droid's clear live-state controls, without copying either product's branding. Shellock should feel distinct from upstream Pi without becoming a separate agent loop.
