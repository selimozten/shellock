# Branding Notes

Shellock's terminal startup should stay text-first and operator-focused:

- Use the double-line console border for the default startup panel.
- Keep product identity, version, mission state, model, context, workspace, and tool contract visible without scrolling.
- Do not render large ASCII art in the default terminal header. Oversized marks belong in docs, release material, or a future optional splash mode.
- Treat user-provided logo/color work as the source of truth once available.

The terminal should feel distinct from upstream Pi without becoming a separate UI shell.
